# lab5-tcp-1

Асинхронные TCP-клиент и TCP-сервер, реализованные на **Qt** с использованием **CMake**.

Этот проект представляет собой лабораторную работу по Qt: настольные приложения **TCP-сервера** и **TCP-клиента** с графическим интерфейсом.  
Приложения работают в **асинхронном режиме**, обмениваются бинарными пакетами через **QDataStream**, имеют разделение по зонам ответственности и покрыты автоматическими тестами.

## Возможности

### Сервер
- Асинхронный TCP-сервер на базе Qt Network
- Задаваемые пользователем IP-адрес и порт
- Запуск и остановка прослушивания из GUI
- Обработка подключения и отключения клиента
- Приём пакетов и формирование ответов
- Логирование сетевых событий в окне приложения

### Клиент
- Асинхронный TCP-клиент на базе Qt Network
- Задаваемые пользователем IP-адрес, порт и timeout
- Ручное подключение и отключение
- Пакетный обмен данными через `QDataStream`
- Логирование сетевых событий в окне приложения
- Три режима обмена:
  1. **Single** — ручная отправка пакетов в рамках долгоживущего соединения
  2. **Long** — периодическая отправка пакетов по таймеру в рамках долгоживущего соединения
  3. **Short** — периодический цикл: подключение → отправка → приём ответа → отключение

## Пакетный протокол

В проекте используется простой бинарный протокол с фреймингом поверх TCP:

- заголовок кадра: `quint32 payloadSize`
- полезная нагрузка: сериализуется через `QDataStream`

### Пакет запроса клиента
- `quint32 number`
- `QString text`

### Пакет ответа сервера
- `quint32 number`
- `QString text`
- `QTime serverTime`

## Архитектура

Код разделён по зонам ответственности.

### Клиент
- `MainWindow` — слой пользовательского интерфейса
- `ClientController` — сокет, таймер, логика режимов работы
- `ClientSettings` — сохранение настроек через `QSettings`
- `PacketProtocol` — сборка/разбор кадров и пакетов
- `SessionMode` — перечисление режимов обмена клиента

### Сервер
- `MainWindow` — слой пользовательского интерфейса
- `ServerController` — серверный сокет, работа с активным клиентом, обработка пакетов
- `ServerSettings` — сохранение настроек через `QSettings`
- `PacketProtocol` — сборка/разбор кадров и пакетов

## Сборка

Пример сборки через CMake:

```bash
cmake -S . -B build
cmake --build build
```

## Запуск

После сборки запускаются оба приложения:

- исполняемый файл сервера
- исполняемый файл клиента

Типичный сценарий работы:
1. Запустить сервер
2. Указать адрес и порт
3. Начать прослушивание
4. Запустить клиент
5. Подключиться к серверу или выбрать один из режимов обмена
6. Наблюдать логи и обмен пакетами в обоих окнах

## Запуск тестов

```bash
ctest --test-dir build --output-on-failure
```

Также можно запускать отдельные тестовые цели через IDE или систему сборки.
